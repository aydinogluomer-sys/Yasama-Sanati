# Components

## Core Principles

- Fewer variants.
- Stronger defaults.
- Local overrides only when the section needs a specific authored feel.

## Standard Component Types

- Button
- Link
- Card
- Section header
- Trust note
- Form field
- Divider

## Component Rules

- Keep button hierarchy simple and recognizable.
- Cards should feel like editorial surfaces, not SaaS boxes.
- Dividers should be hairline-level, subtle, and structural.
- Form fields should feel calm and readable.
- Trust notes should be visible enough to matter.

## Anti-Component Habits

- Avoid one-off patterns that cannot scale.
- Avoid scattering special-case styling across many files.
- Avoid adding new component states without reason.
